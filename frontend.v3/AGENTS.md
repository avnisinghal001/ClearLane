# AGENTS.md — Frontend v3 (`frontend.v3/`)

> Scope: the **role-based TypeScript app** for the v3 (H3 cell) stack. Read the root
> `AGENTS.md` first — the honesty contract governs all UI copy and the three-number
> separation governs how operational state is shown. This is the **successor** to the
> JSX `frontend/`; it is **not the deployed UI yet** (`vercel.json` still builds
> `frontend/`). Don't assume changes here ship to production.

## Stack

- **React 18 + Vite 5 + TypeScript** (strict-ish; `npm run build` runs `tsc --noEmit`).
- **shadcn/ui** primitives (Radix + `class-variance-authority` + `tailwind-merge`) in
  `src/components/ui/`; **TailwindCSS** (`tailwind.config.js`, `src/index.css`).
- **react-router-dom v6** (real routes, not hash), **leaflet** + `leaflet.heat` for
  maps, **recharts** for charts, **@tanstack/react-table** for tables, `lucide-react`
  icons. `@` aliases `./src`.

```bash
cd frontend.v3 && npm install
cp .env.example .env          # VITE_API_BASE empty -> same-origin /api (Vite proxy in dev, fn in prod)
npm run dev                   # :5173  (Vite proxies /api -> VITE_BACKEND_PROXY, default :8000)
npm run build                 # tsc --noEmit && vite build -> dist/
npm run demo                  # python scripts/build_demo_v3.py -> public/demo-v3/*.json
```

## Three role apps (`src/App.tsx`, `src/roles/`)

`App.tsx` is a `BrowserRouter` with **route-level code splitting** (each role lazily
imports its heavy deps — leaflet/recharts — so the initial bundle stays lean):

| Route | Role | Guard | App |
|---|---|---|---|
| `/` | landing | — | `RoleLanding` |
| `/citizen` | citizen | open | `roles/citizen/CitizenApp` (report a problem, my reports) |
| `/police` | station | `RequireRole role="station"` | `roles/police/PoliceApp` (hotspots, tickets, dispatch) |
| `/govt` | govt | `RequireRole role="govt"` | `roles/govt/GovtApp` (KPIs, scorecard, analytics, stations) |

`RequireRole` reads `getAuth()` (`src/lib/auth.ts`) and redirects to `/` if the role
doesn't match. Auth talks to `/api/auth/*` (force-command RBAC in
`api/clearlane/force.py`): roles are `station` and `govt`.

## The offline-first contract (do not break this)

**The app must always render — even with no backend and no internet** (judging
safety). `src/lib/api.ts` is the single API layer:

- Every **read** tries the live backend (`VITE_API_BASE + /api/v3/...`, or the Vite
  proxy / Vercel fn at same-origin) and on **any** failure falls back to a bundled
  `public/demo-v3/*.json`. On failure it flips `LIVE = false` (subscribe via
  `onLiveChange`); the UI shows a "DEMO (offline)" state.
- Every **write** (complaint / ticket / resolve / officer-feedback) tries the backend
  then falls back to `src/lib/localStore.ts` — the **in-memory mirror of the v3 H3
  rules** in `api/clearlane/v3.py` (`OP_RULES`, `V3_REASONS`, snap radius, decay,
  clamp). **If you change those rules on the backend, change `localStore.ts` to
  match** or the offline demo diverges.
- The demo bundle in `public/demo-v3/` is generated by `scripts/build_demo_v3.py`
  (`npm run demo`) — **do not hand-edit it**. New data dependency = full chain:
  `ml.v3` stage emit → `api/clearlane/v3.py` route → `api.ts` reader + demo fallback.

## Layout of `src/`

| Path | What |
|---|---|
| `lib/api.ts` | offline-first `/api/v3` client (the only place network access lives) + `isLive`/`onLiveChange`. |
| `lib/localStore.ts` | in-memory mirror of the v3 backend write rules (offline loop). |
| `lib/auth.ts` | session token storage + `getAuth()` for `RequireRole`. |
| `lib/types.ts` | shared TS types (`Cell`, `Ticket`, `Station`, `MapPayload`, `When`, …). |
| `lib/time.ts` | day-of-week helpers (`dowForWhen`, `dowLabel`) — **date granularity only**. |
| `lib/constants.ts`, `lib/format.ts`, `lib/utils.ts` | config constants, formatting, `cn()` class merge. |
| `hooks/` | `useConfig` (fetches `/api/config` map keys), `useMapData`, `useGeolocation`, `useMediaQuery`. |
| `components/map/` | `ClearLaneMap` + Base/Heat/Traffic layers; `mapmyindia.ts` SDK loader, `pin.ts`. |
| `components/ui/` | shadcn primitives — reuse these, don't hand-roll buttons/dialogs. |
| `components/` | `AppShell`, `Kpi`, `DataTable`, `LiveBadge`, `SourceBadge`, `TimeControl`, `LoginDialog`, `toast`, … |
| `roles/<role>/` | the per-role screens (above). |

## Maps (`components/map/`)

Three engines, in order: **(1)** Mappls `map_load` (REST key), **(2)** Mappls Web SDK
v3 (static key), **(3)** OSM/Carto tiles fallback. Keys come from `useConfig` →
`GET /api/config` at runtime (server-provided), or `VITE_MAPMYINDIA_KEY` baked into
the build. If both are absent the map falls back to OSM/Carto — keep that fallback
working so the demo never shows a blank map.

## Honesty rules in the UI

- Never label anything "congestion" measured from data. `congestion_source` is
  `modeled` / `mappls_typical` / `live` — surface that label; the data is tickets.
- Keep the three numbers visibly distinct (historical / live adjustment /
  operational) anywhere a cell's priority is shown.
- Time is **day-of-week only** (the ticket timestamp is upload time, not parking
  time). Don't introduce an hour-of-day control implying measured hourly traffic.

## Conventions

- Keep the per-route bundle lean — respect the lazy-loading boundary; don't import a
  role's heavy deps at the top level of `App.tsx`.
- Reuse `components/ui/` primitives and the `@/` alias. Match the existing Tailwind +
  shadcn idiom (no ad-hoc CSS files).
- `npm run build` must pass `tsc --noEmit` — keep it type-clean.
