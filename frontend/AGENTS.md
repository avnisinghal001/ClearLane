# AGENTS.md ŌĆö Frontend (`frontend/`)

> Scope: the React command-center dashboard. Read the root `AGENTS.md` first ŌĆö the
> honesty contract applies to all UI copy, and the three-number separation governs
> how operational state is displayed.

## Stack

- **React 18 + Vite 5**, plain JSX (no TypeScript, no router lib ŌĆö hash-based routing).
- **react-leaflet / leaflet** for maps, **recharts** for charts, **qrcode.react**
  for local QR (no remote QR service).
- No CSS framework ŌĆö a single hand-written stylesheet at `src/styles/app.css`.

```bash
cd frontend && npm install
cp .env.example .env          # VITE_API_BASE=http://localhost:8000
npm run dev                   # http://localhost:5173
npm run build                 # -> dist/  (bundle is ~120KB gz; keep it lean)
```

## The offline-first contract (do not break this)

**The dashboard must always render ŌĆö even with no backend and no internet.** This
is a judging-safety requirement.

- `src/lib/api.js` is the single API layer. Every read call tries
  `VITE_API_BASE + path`, and on any failure falls back to a bundled
  `public/demo/*.json` file. When a call fails it flips `LIVE = false` and the
  header badge shows "DEMO (offline)".
- The operational write loop (`opSnapshot`, `opComplaint`, `opDispatch`,
  `opFeedback`, `opPatchStatus`) tries the backend, then falls back to
  `src/lib/localOps.js` ŌĆö an **in-memory mirror of `backend/app/operational.py`'s
  exact rules** (OP_RULES, BBOX, nearest-zone-600m, decay, clamp). If you change
  those rules on the backend, change `localOps.js` to match.
- The demo bundle in `public/demo/` is regenerated by the ML pipeline's
  `run_all.py` (do not hand-edit it). If you add an artifact the UI needs, make
  the pipeline emit + bundle it, add a backend route, and add a demo fallback in
  `api.js`.

## App shell & routing (`src/App.jsx`)

- State lives in `App`: `payload` (map_payload), `snapshot` (operational layer),
  `view`, `selected` zone, `filter`, `flyTo`, mobile flag.
- It loads `/api/map/payload` once, seeds the offline zone index
  (`seedOpZones`), and **polls the operational snapshot every 5s** so the command
  center visibly updates.
- Routing is **hash-based**:
  - `#/dispatch/{id}` ŌåÆ standalone mobile `Dispatch` screen;
  - `#/officer` (or any phone-width screen unless `#/dashboard`) ŌåÆ `OfficerView`;
  - otherwise the full dashboard with the nav `VIEWS`.
- `VIEWS` = Command Map, Today / Emergency, Priority Queue, Flow Impact,
  Repeat Offenders, Operations Loop, Timing Gap, Coverage/ROI, Forecast, Typology,
  Station Command, Methodology & Validation.

## Libraries (`src/lib/`)

- `api.js` ŌĆö API + offline fallback (above). All network access goes through here.
- `localOps.js` ŌĆö offline mirror of the backend operational rules.
- `plain.js` ŌĆö **plain-language translation layer** for the officer view. Maps
  tierŌåÆ"Top priority", pressureŌåÆ"blocks traffic constantly", responsiveness/
  habitual/blind-spot ŌåÆ plain sentences, plus km/ago/haversine helpers. **No raw
  scores or jargon are shown on the officer view** ŌĆö route them through `plain.js`.
- `format.js` ŌĆö number/label formatting helpers.

## Honesty rules in the UI

- Never label anything as "congestion" measured from data. Evening gap = coverage
  gap vs *assumed* peaks; hour-slider activity = **recorded enforcement activity,
  not traffic** (label it as such ŌĆö e.g. 10:00 = many zones active, 19:00 Ōēł empty).
- In the zone drawer / operations views, keep the three numbers visibly distinct:
  historical priority (immutable) vs live adjustment vs operational priority.
- The validation slider is a real client-side re-rank from the A/B/C pillar
  scores, labelled exploratory, resettable to 50/30/20. Keep it real ŌĆö do not
  reintroduce a fake/dead control.

## Conventions for new work

- Keep the gzip bundle lean (~120KB). Avoid heavy deps.
- Any new data dependency needs the full chain: pipeline emit ŌåÆ demo bundle ŌåÆ
  backend route ŌåÆ `api.js` fallback ŌåÆ component.
- Mind React hooks order (a past bug was a conditional-hook ordering issue in
  `CoverageSimulator`); declare all hooks before any early return, as `App.jsx` does.
- Match the existing terse JSX style and the single `app.css` stylesheet.
